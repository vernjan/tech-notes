# SSO

- single-sign on
- 2 main protocols
    - **SAML**
        - older, enterprise-heavy standard based on XML
        - best for legacy enterprise systems, corporate intranets, and B2B SaaS applications
    - **OIDC**
        - modern, lightweight identity layer built on top of OAuth 2.0
        - best for modern web applications, mobile apps, and APIs
        - behind "Log in with Google" or "Log in with Apple"
        - uses **JWT** (JSON Web Token)
        - extension of OAuth
            - **OAuth** - authorization
            - **OIDC** - authentication
